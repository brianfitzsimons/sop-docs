Multi-Chart Retrieval — Step-by-Step Call Flow

Follow each step in order. Decision points are clearly marked — act on whichever applies and continue to the next step.

Retrieval Operations · Offsite Updated 03/11/2026 Multi-ORG / Multi-Health Plan / High Volume
  1. 1
    Before the call
    Pre-Call Preparation
    • Open all active ORGs for this site in ChartFinder
    • Add up the "to go" chart count across every ORG — note the combined total
    • Note how many health plans are involved (1 or 2+)
    • Check TT notes — confirm whether address, fax, and members/providers were previously verified and still match current demographics
    • Note the inventory status of each ORG: Unscheduled / Past Due / Scheduled / Rescheduled
  2. 2
    Pre-call check
    Check Demographics — Do You Need to Re-verify?
    Decision — Do TT notes match current demographics?
    YES — match No re-verification needed. Proceed directly to Step 3. (Update EMR field only if it's blank — this does not require full reverification.)
    NO — changed Full reverification required on this call: address, fax, member/provider names, and EMR system.
    Also reverify if: PHI will be released and the HIH vendor name is not listed in notes.
  3. 3
    Call opening
    Open the Call
    "Hello, this is [Your Name] calling on behalf of [Health Plan name / multiple health plans] from Datavant. Please be advised this call is being recorded."
    • If 1 health plan → name it specifically
    • If 2+ health plans → say "multiple health plans" — do not name both
    • State the project type: MRA / ACA / RADV / HEDIS
    • If the inventory is Past Due, Scheduled, or Rescheduled → briefly review last 2–3 call outcomes before opening
  4. 4
    Verification
    Verify the Site
    Decision — Is full verification needed?
    DEFAULT Verify 2 members (name + date of birth) for this phone number.
    EXCEPTION If site refuses member verification → verify 2 providers (full name including suffix, e.g. John W. Smith Jr.) — document this explicitly in TT notes.
    Multi-ORG tip: The same provider name can be used across multiple ORGs if that provider appears in all the relevant ORGs.
  5. 5
    Chart discussion
    Discuss All ORGs and Total Chart Count
    • Disclose the total: "We are requesting [X] charts across [Y] requests."
    • Cover every ORG in this call — do not skip any
    • For Unscheduled: discuss chart count and request type, send Provider Package
    • For Past Due / Scheduled / Rescheduled: discuss charts received vs. to go, confirm PP receipt, offer shipping support if needed
    • If the site has multiple ORGs each under 15 charts individually → state the combined total: "Combined we have [X] charts — let me explain what we can offer."
  6. 6
    Retrieval method
    Select the Retrieval Method — Use Combined Chart Count

    Always use the combined total across all ORGs — not individual counts — to determine the method.

    Combined Chart Count Offer in This Order
    299+ charts Do not schedule. Escalate via SPI Intake job aid.
    60–299 charts Onsite (CRS) → SFTP → EMRR → Portal → Fax → Mail
    15–59 charts Onsite (CRS) → EMRR → Portal → Fax → Mail → Email
    Under 15 charts Portal → EMRR → Fax → Mail → Email → Onsite*
    * If site insists on Onsite with under 15 charts → submit a CRS Request for approval first. Do not schedule without approval.
    Decision — 15+ charts and site agrees to CRS (Onsite)?
    YES Estimate time: total charts ÷ 10 = hours needed. Schedule minimum 3 business days out. Ask for any site forms or special instructions.
    NO Offer the next method in the hierarchy above. Confirm retrieval details and access requirements with the provider.
  7. 7
    Commitment date
    Negotiate Turnaround Time
    • Reference the Expected Schedule Target Date table below
    • If provider agrees to the target date → thank them, confirm and log it
    • If they cannot meet the target → "Can you provide a reason for the delay so we can advise the health plan(s)?"
    • HEDIS & RADV: always use a 5 business day completion target
    Chart CountTarget Completion
    1–2 charts2 business days
    3–5 charts3 business days
    6–10 charts4 business days
    10+ charts5 business days
    Commit date rules: ≤8 business days → set Schedule Date = commit date. >8 business days → set to actual commit date and document the reason in the summary note. Maximum 30 business days.
  8. 8
    Before hanging up
    Confirm It's a Valid Schedule

    ✓ Valid — ALL must be true

    • Spoke to a live person at the site
    • Commitment date in TT notes
    • Commitment date in ChartFinder
    • Provider Package sent (if required)

    ✗ Not valid — do not log as scheduled

    • IVR or website only — no live contact
    • Left voicemail, no callback
    • Date not documented in both places
    Soft schedules are invalid as of 03/11/2024. Must have live confirmation.
  9. 9
    Documentation
    Log and Close — Do This for Every ORG Worked
    • Log the commitment date in TT notes and in ChartFinder for all ORGs covered in this call
    • Update the retrieval method for each ORG
    • Update EMR field if blank (does not require reverification)
    • Send Provider Package if: site is unscheduled, chart volume changed, or PP was never received
    • If commit date is >8 business days → document the reason clearly in the summary note
    • For CRS: log site forms and special instructions in TN Note; forward forms via CCL to Christine Avery Smith
  10. VM
    If no live contact reached
    Leave a Voicemail
    "Hello, this is [Your Name] calling on behalf of multiple health plans from Datavant today regarding a chart retrieval request. Please return my call at 877-445-9293 with the outreach ID [Outreach ID] at your earliest convenience. Again, the number is 877-445-9293 and the reference site ID is [Outreach ID]. Thank you, have a great day!"
    • Speak slowly — repeat the callback number and outreach ID twice
    • If single health plan → substitute "multiple health plans" with the specific plan name
    • Document the voicemail attempt in TT notes — this is not a valid schedule
  11. ✓
    Call Complete — All ORGs Covered

    All ORGs documented · Commitment date logged in notes and ChartFinder · Provider Package sent if required · EMR field updated
